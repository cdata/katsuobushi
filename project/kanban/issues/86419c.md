## What to build

From the 55cfca review: `Renderer` resolves its `color` bool from `stdout().is_terminal()` (output.rs:resolve), but streamed reports now write to **stderr** via `emit_progress`. So in a rare mixed-redirect (stdout piped, stderr a TTY, or vice-versa) the color decision keys on the wrong stream. Every real path is correct today (backgrounded => stdout not a TTY => plain lines, the intended target; interactive => same TTY => colored), so this is a minor correctness-hygiene nit, not a live bug.

## Fix direction

Let the streamed-progress path base its color decision on **stderr**s TTY-ness (the stream it actually writes to), e.g. resolve a separate progress-color bit from `stderr().is_terminal()`, or thread the target stream into the color resolution. Keep the non-streaming `emit` keyed on stdout.

## Acceptance criteria

- [ ] Streamed reports color-gate on the stream they write to (stderr); `emit` unchanged (stdout).
- [ ] Tests cover the mixed-redirect decision; cargo test + clippy -D warnings green.
