## What happened

On `cdata/arcanabasis` (dogfooding board, 2026-07-20), `project status set
<id> accepted` repeatedly appended a **new** `***` + `## Archive` section
immediately before `%% kanban:settings` instead of appending to the existing
Archive lane. The board accumulated four `## Archive` sections over a few
sessions. A card stranded in a duplicate lane then breaks the tool:
`status set` on it fails with `error: card <id> is in an unrecognized lane`,
and the card silently vanishes from `project status` listings.

## Reproduction

1. A board whose single `## Archive` heading is NOT preceded by the `***`
   separator — which is exactly what `prettier`-style markdown formatting
   produces (the repo convention `markdown format` runs over kanban docs
   and normalizes/removes the thematic-break separator; two sandbox agents'
   format-only chore commits did this, diff `4 +--- / 1 insertion(+),
   3 deletions(-)` on BOARD.md).
2. `project status set <id> accepted` on any Ready card.
3. Result: a fresh `***\n\n## Archive\n\n- [x] [[<id>]]` block appended at
   the bottom; the original Archive lane untouched.

Confirmed deterministic: after hand-merging the board back to one Archive
lane (without `***`), the very next accept reproduced the duplicate.
Restoring the `***` before `## Archive` makes appends land correctly.

## Expected

- Archive locating should tolerate a missing/normalized `***` separator
  (match the `## Archive` heading itself, or re-insert the separator once),
  never append a second Archive section.
- `project lint` should flag duplicate lane headings — it reported
  `clean — board and notes are consistent` throughout the corrupted state,
  and only surfaced a symptom later (`stale-disposition`) via an unrelated
  path. Cards in unrecognized lanes should also be lint findings, since
  they disappear from `status` output entirely.

## Acceptance criteria

- [ ] Accepting a card on a board whose Archive heading lacks the `***`
      separator appends into the existing Archive lane
- [ ] `project lint` errors on duplicate lane headings and on cards in
      unrecognized lanes


## Update (2026-07-20)

Sharper root cause: the markdown formatter is only one of two separator
killers. The `project` CLI's **own full-board rewrite** on an ordinary
status move also drops the `***` before `## Archive` — observed directly
after a `sandbox dispatch` claim rewrote the board, with no formatter run
in between. The CLI destroys its own archive anchor, so the duplicate-lane
append reproduces even on a formatter-free board. See [[3e9510]] for the
broader formatter-stability churn this belongs to.
