## What happened

On `cdata/arcanabasis` (2026-07-19/20), **three separate sandbox agents in
one day** each had their lint gate blocked by "pre-existing markdown format
drift" in `project/kanban/BOARD.md` and each committed a format-only chore
to fix it — and the drift returns after every board mutation, because the
`project` CLI's board writer emits markdown that standard formatters
(prettier-style, as run by the repo's `markdown format` gate) rewrite:

- Empty lanes are written with a double blank line (`## Needs Review\n\n\n##
  Ready`); formatters collapse to one.
- The `***` thematic-break separator before `## Archive` is dropped by the
  CLI's own full-board rewrite on ordinary status moves (observed
  immediately after a `dispatch` claim rewrote the board — no formatter
  involved). Losing the separator then triggers the duplicate-Archive
  append bug (see [[5b4df3]]).

The result is a permanent churn loop: CLI mutation → formatter-dirty board
→ agent commits a "fix drift" chore → next CLI mutation reverts it. Every
dispatched card pays the tax, and the noise buries real diffs in board
history.

## Expected

The board writer should emit formatter-stable markdown: single blank lines
between lanes, and either preserve the `***` separator across rewrites or
stop depending on it (per [[5b4df3]]). A board round-tripped through the
CLI and then through prettier should be byte-identical.

## Acceptance criteria

- [ ] `project status set` / `new` / `prioritize` rewrites are
      byte-stable under a prettier-style markdown formatter
- [ ] The archive separator (or its replacement anchor) survives every
      CLI rewrite
