## What to build

Merge `project show` into `project status`, removing `show` ENTIRELY (no alias — we are the only consumer). Bare-positional layout:

- `project status` — list the whole board (active lanes in priority order, then the archive); todays bare `project show`.
- `project status <id>` — detail one card; todays `project show <id>`.
- `project status --available` — grabbable filter.
- `project status --lane <lane>` — filter to one lane. This is todays `show --status <x>`, RENAMED to `--lane` (since `status --status` reads absurdly).
- `project status --json` — machine-readable, everything.
- `project status set <id> <to> [--force]` — unchanged mutate path.

Implementation notes:
- In clap, `status` gets an optional `[id]` positional + the `set` subcommand + the filter flags, via `#[command(args_conflicts_with_subcommands = true)]`. Card ids are 6-hex so none can equal the literal `set` — no ambiguity.
- Remove the `Show` ProjectCommand variant + its args; move the list/detail/filter wiring under `Status`. query.rs `list_all`/`show_one` stay; only CLI dispatch changes.
- Update the `project` SKILL.md and any README / help text referencing `project show`.
