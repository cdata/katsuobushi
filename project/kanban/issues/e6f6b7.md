## What happened

The archive parse now anchors on the `## Archive` heading (card 5b4df3),
tolerating any/missing thematic-break separator. A side effect: a board
with a bare `***`/`---` separator followed by `- [x]` items but **no**
`## Archive` heading is no longer read as an archive. The old parser set
`archive_present` on a bare separator and swallowed the following items
into the archive; the new parser treats the separator as a stray line and
pushes those items onto the last active lane (or drops them if none
precedes) — un-archiving cards into live work, or losing them.

## Notes

Neither Obsidian-Kanban nor this tool ever writes a separator without a
`## Archive` heading, so real-world impact is low. Filed as a tolerance
regression from the heading-anchored rewrite, for completeness.

## Acceptance criteria

- [ ] A board with `<sep>` + `- [x]` items and no `## Archive` heading
      either archives those items or `lint` flags the malformed shape,
      rather than silently re-laning/dropping them
