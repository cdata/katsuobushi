## What happened

A sandboxed agent building a Bevy-scale Rust workspace's test derivation
(`cargo test --no-run` inside a `nix build` check) failed with:

```
error: failed to write to `/build/source/target/debug/deps/rmeta1tWPEN/full.rmeta`:
No space left on device (os error 28)
```

while `df -h` inside the guest showed every provisioned volume nearly
empty: the store overlay (`/dev/vda`, 16G) at 2%, `/scratch` (`/dev/vdc`,
32G) at 30%, the nix db volume at 5%. Observed 2026-07-19 on
`cdata/arcanabasis` (vcpu 8, mem 12288, storeVolumeSize 16384,
scratchVolumeSize 32768).

## Diagnosis

The nix-daemon's sandboxed build directory (`/build` inside a build) lives
on the guest's **root filesystem, which is a RAM-backed tmpfs sized at half
of `mem`** (~6G on a 12G guest). None of the `lib.sandbox` volume knobs
(`storeVolumeSize` / `scratchVolumeSize` / `dbVolumeSize`) touch it, so
consumers cannot fix this by resizing anything the API exposes. The failed
build's tree is cleaned up on failure, so post-mortem `df` shows an
almost-empty rootfs â€” which makes the failure look like a phantom and cost
two agent sessions misdiagnosing it (first as a too-small store overlay).

Any derivation whose build tree exceeds ~mem/2 fails regardless of volume
provisioning, and big tmpfs build trees also compete with builder RAM.

## Suggested fix

Default the guest's nix build scratch onto the disk-backed scratch volume,
e.g. in the guest module:

```nix
nix.settings.build-dir = "/scratch/nix-build";   # TMPDIR on nix-daemon for older Nix
systemd.tmpfiles.rules = [ "d /scratch/nix-build 0755 root root -" ];
```

and document that `scratchVolumeSize` must then cover cargo/XDG caches
plus transient nix build trees (tens of GB for Bevy-scale workspaces;
volumes are sparse so generous defaults are cheap). A dedicated
`buildVolumeSize` would also work if mixing concerns on `/scratch` is
undesirable.

## Workaround (current consumers)

```nix
guestModules = [{
  nix.settings.build-dir = "/scratch/nix-build";
  systemd.tmpfiles.rules = [ "d /scratch/nix-build 0755 root root -" ];
}];
```

plus a larger `scratchVolumeSize`.

## Acceptance criteria

- [ ] A derivation with a build tree larger than mem/2 (but smaller than
      the scratch volume's free space) builds successfully in-guest
- [ ] The sizing interaction is documented on the volume knobs
