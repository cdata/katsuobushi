## What to build

From the 9f44c7 review: the shipped fetch check only catches the dirty-tree stash seed (`WIP on …`). A clean-tree seed is HEAD and a resumed instance seed is the prior branch commit — both real commits — so an agent that ends without committing on those dispatches is NOT flagged. Make the check robust and false-positive-free.

## Acceptance criteria

- [ ] persist the seed commit SHA (computed in `sandbox/start.rs::resolve_seed`) into the instance (instance.json / the `Instance` struct)
- [ ] `sandbox fetch` reads it and compares `git rev-parse sandbox/<inst>` (tip) against the seed SHA; `landed = tip != seed`
- [ ] covers all three seed kinds (dirty stash, clean HEAD, resume); drop the subject-heuristic
- [ ] tests for each seed kind
