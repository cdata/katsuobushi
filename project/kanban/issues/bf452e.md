## What happened

`to_text` unconditionally writes `preamble.trim_end()` followed by
`"\n\n"`. When a board has no frontmatter, the preamble is empty, so the
output begins with two blank lines before the first `## ` lane. A
prettier-style formatter strips leading blank lines, so this output is not
formatter-stable — contradicting the `to_text` byte-identity guarantee
(the property card 3e9510 established) for the empty-preamble shape.

## Reproduction

Parse `## To-do\n\n- [ ] [[a3f7b2]]\n` (no frontmatter) and re-emit: the
result starts with `\n\n## To-do…`.

## Notes

Low severity: `project init` always writes frontmatter, so real boards
carry a non-empty preamble. This is a robustness gap for hand-created or
corrupted boards, not a live incident.

## Acceptance criteria

- [ ] `to_text` on a frontmatter-less board emits no leading blank lines
- [ ] The emitted board is byte-stable under the `markdown format` gate
