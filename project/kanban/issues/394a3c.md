## What to build

Three behaviors on the merged `project status` surface (blocked_by the merge card — build on its final shape).

### 1. `project status set --accept-all`
Shorthand: move EVERY card in the Ready lane to Accepted (ready->accepted).
- Make `set` positional `id`/`status` optional; add `--accept-all`. Validate exactly one of {id+status, --accept-all}.
- Iterate the Ready lane, applying ready->accepted per card via `set_status::apply` (archives + stamps disposition + disposition_at below).
- Print each accepted id (human) / JSON list (--json).
- This is a product-owner action; the human-only accept rule still holds (a human runs it) — no code enforcement needed.

### 2. `disposition_at` timestamp (the "accepted on" field)
When a card crosses into a TERMINAL state (accepted OR cancelled), stamp a `disposition_at:` RFC-3339 timestamp in note frontmatter (paired with the existing `disposition:`), via the project Clock seam. Clear it on a forced reopen out of terminal (mirror how `disposition` clears). Thread a `Clock` into `set_status::apply` (prod SystemClock; tests FixedClock). Add `disposition_at` to `NoteMeta`.

### 3. 24h archive window in the list view
In `project status` (non-JSON list), hide archived cards whose `disposition_at` is older than 24h (missing -> treat as old, hidden). `--json` UNAFFECTED (all archived returned). Applies to ALL archived (accepted AND cancelled). Use the Clock for now.
