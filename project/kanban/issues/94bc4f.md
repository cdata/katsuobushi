## What to build

Make `katsuctl` compile on non-Linux (macOS) via conditional compilation. The `project` domain is cross-platform; only the `sandbox` domain is Linux-only (tokio-vsock / QMP). On non-Linux, `katsuctl` builds with `project` (+ shared bits) and omits `sandbox`.

Chosen approach: **pure `#[cfg(target_os = \"linux\")]`** (NOT a cargo feature).

### 1. Decouple the shared renderer
`project/` only borrows `sandbox::output` (Renderer / render_table / TableCell / Reported — pure rendering). Move `src/sandbox/output.rs` to top-level `src/output.rs` (always compiled) and rewire every `crate::sandbox::output::*` import (7 files under `project/`, plus sandbox’s own uses) to `crate::output::*`. After this, `grep -r \"use crate::sandbox\" src/project/` MUST be empty.

### 2. Target-gate the sandbox deps + code
- In `Cargo.toml`, move the Linux-only deps (`tokio-vsock`, `katsuobushi-sandbox-protocol`, and anything else the sandbox domain needs that does not build on macOS — check `tokio`, `nix`, etc.; keep genuinely cross-platform deps ungated) into `[target.'cfg(target_os = \"linux\")'.dependencies]`.
- Gate `mod sandbox`, the `Domain::Sandbox` variant + its clap args + `sandbox dispatch`, behind `#[cfg(target_os = \"linux\")]` in `main.rs`.
- The cross-platform path (`output`, `project`, main’s project arm) must reference NO Linux-only symbol.

### 3. Flake: build katsuctl everywhere
- Build `controlCrates.katsuctl` on ALL systems (currently isLinux-only). On non-Linux the crate cfg’s out sandbox automatically; ensure the crane build has no tokio-vsock in the darwin dep closure.
- Move `project` menuCommands, the on-PATH `katsuctl`, and `lib.project` OUT of the `isLinux` guards (cross-platform now). Keep sandbox apps / checks / menu Linux-only. Expose `packages.<system>.katsuctl` on non-Linux too.

## Commits
Commit in logical steps (Conventional Commits): (1) extract the output module, (2) target-gate the sandbox domain, (3) flake cross-platform build.

## Acceptance criteria
- [ ] Linux `cargo test -p katsuobushi-controller` stays green; clippy -D warnings clean; cargo fmt clean.
- [ ] `grep -r \"use crate::sandbox\" rust/katsuobushi-controller/src/project/` is EMPTY.
- [ ] `mod sandbox` / `Domain::Sandbox` / `sandbox dispatch` all behind `#[cfg(target_os = \"linux\")]`.
- [ ] Cargo.toml target-gates the Linux-only deps (no tokio-vsock outside the linux target).
- [ ] Flake exposes `packages.aarch64-darwin.katsuctl` + `packages.x86_64-darwin.katsuctl` and `lib.project` cross-platform (verify via `nix eval`, since darwin cannot BUILD here); sandbox apps/checks stay Linux-only.
- [ ] `nix flake check` green on Linux.

NOTE: the macOS *compile* cannot be validated from the Linux sandbox (cfg(target_os) follows the host). Ensure correctness by cfg discipline (no Linux-only symbol in the un-gated path). Final darwin build is the owner’s.
