plain git clone seeded from the host's current state — use `git`, never `jj`.

## Build & test (the gate)

Everything runs through the Nix dev shell. **One simple command per Bash call**
(no `cd`, no `&&`/`;`, no pipes/redirection):

- Build/test: `nix develop -c cargo test -p katsuobushi-controller`
- Lint gate (must be clean): `nix develop -c cargo clippy -p katsuobushi-controller --all-targets --all-features -- -D warnings`
- Format: `nix develop -c cargo fmt -p katsuobushi-controller`
- If you touch any Markdown (e.g. a `SKILL.md`): format then lint it —
  `nix develop -c markdown format`, then `nix develop -c markdown lint` (must be
  clean; the docs gate runs Prettier).

The **first `cargo` build cold-compiles the crate and can take many minutes** —
start it ONCE in the background and wait for it; never launch a second build
while one is in flight (they only contend on the cargo lock).

`clippy -D warnings` green and `cargo test` green are the acceptance gate.
Run `cargo fmt` before committing (a hard gate).

## Conventions

- Read/search files with your Read/Grep/Glob tools, not shell commands.
- The `project` domain lives in `rust/katsuobushi-controller/src/project/`; the
  `sandbox` domain in `.../src/sandbox/`. Pure logic is unit-tested; verbs are
  thin IO glue over an `Fs` seam.
- Match the surrounding code's style, comment density, and BDD-style test names
  (`it_*` / descriptive `snake_case`).

## Finishing

- Commit on your branch with a Conventional Commit
  (`type(scope): Sentence-case summary`, subject < 70 chars, **no
  `Co-Authored-By` trailer**). Then `git push`.
- Run `report done "<one-paragraph summary of what you built and how you proved
  it — tests/clippy green>"`. If genuinely stuck: `report blocked "<what you
  need>"`. Never end your turn with a background build still running, and never
  leave work uncommitted.
